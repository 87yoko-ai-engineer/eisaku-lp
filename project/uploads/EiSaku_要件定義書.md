# EiSaku — 英作文添削ウェブサイト 要件定義書

## 1. 概要

**EiSaku（エイサク）** は、英語学習中の日本人社会人向けの英作文添削ウェブサイトです。
ユーザーが書いた英文に対し、Claude API を活用して多角的な添削とフィードバックを返します。
単発の添削だけでなく、**レベル別に段階的にステップアップ**できる学習体験を提供することが特徴です。

まずは個人利用を前提にPC版のプロトタイプを作成します。

---

## 2. ターゲットユーザー

- 英語学習中の日本人社会人（想定レベル：TOEIC 600〜800点）
- ビジネスメール・レポート・エッセイなど、実務で使える英作文力を身につけたい層
- **初期リリースではオーナー本人のみが利用**（認証・課金・複数ユーザー対応は対象外）

---

## 3. コアコンセプト

1. **多角的な添削**：文法だけでなく、語彙・文体・論理構成まで指摘
2. **レベル別ステップアップ**：ユーザーの現在地に合わせて、少しずつ難易度が上がる文法・構文を含む課題を提示
3. **振り返りの仕組み**：頻出ミスを可視化し、同じ間違いを繰り返さないよう促す
4. **ゲーミフィケーション**：レベル・バッジで継続学習のモチベーションを維持

---

## 4. 機能要件

### 4.1 添削機能（コア機能）

- ユーザーが英文を入力する
- Claude API が以下の観点で添削：
  - 文法ミス（Grammar）
  - スペル・句読点（Spelling & Punctuation）
  - 語彙の選択（Word Choice / Naturalness）
  - 文体・トーン（Tone：ビジネス/カジュアル）
  - 論理構成・文章全体の流れ（Structure & Flow）
- 添削結果は「原文」「修正文」を**並列表示**
- 修正箇所は**ハイライト**され、クリックまたはホバーで**「なぜ修正したか」の解説ポップアップ**が表示される
- 各修正に観点タグ（Grammar / Vocabulary / Tone など）が付与される

### 4.2 学習モード（2種類）

**A. Free Mode（自由添削）**
- ユーザーが任意の英文を書いて添削してもらう
- 業務メールや日記など、自分の目的で使いたいときに使用

**B. Challenge Mode（レベル別課題）**
- 現在のレベルに応じたお題（Writing Prompt）が提示される
- レベルが上がるごとに、新しい文法項目・構文が課題に含まれる
  - 例：Lv.1 基本5文型 → Lv.2 過去形・現在完了 → Lv.3 関係代名詞 → Lv.4 仮定法 → Lv.5 ビジネス定型表現
- 課題には「使ってほしい文法・構文」のヒントが表示される
- 添削結果で「ヒントの文法が正しく使えているか」もチェックされる

### 4.3 進捗・ゲーミフィケーション

- **レベルシステム**：Challenge Mode のクリアや添削回数に応じてレベルアップ
- **バッジ**：特定条件達成で獲得（例：「連続5日学習」「関係代名詞マスター」「100語チャレンジ達成」）
- **頻出ミスランキング**：これまで指摘された誤りを観点別・文法項目別に集計してランキング表示
  - どの分野に弱点があるかを可視化
  - クリックすると過去の該当ミス一覧を参照できる

### 4.4 履歴機能

- 過去に添削された英文を日付順で一覧表示
- 各エントリをクリックすると添削結果を再表示
- レベル・観点タグでフィルタリング可能

## 5. 画面構成

| # | 画面名 | 役割 |
|---|--------|------|
| 1 | トップページ | サービス紹介・Free/Challenge Mode への導線 |
| 2 | 添削画面（Free Mode） | 自由英作文の入力と添削結果表示 |
| 3 | 添削画面（Challenge Mode） | レベル別お題＋英作文入力＋添削結果表示 |
| 4 | 履歴ページ | 過去の添削一覧 |
| 5 | マイページ | レベル・バッジ・頻出ミスランキング |

---

## 6. 画面ごとの詳細

### 6.1 トップページ

- 上部：サービス名「EiSaku」ロゴとキャッチコピー
  - 例：「書いて、直して、伸びていく。あなた専用の英作文コーチ」
- 2つのモードを大きめのカードで提示：
  - 「✍️ Free Mode — 自由に書く」
  - 「🎯 Challenge Mode — レベルに挑む」
- 直近の学習状況サマリー（現在のレベル、連続学習日数、最新バッジ）
- グローバルナビ：トップ / 添削 / 履歴 / マイページ

### 6.2 添削画面（Free Mode）

- **レイアウト**：横2分割（左：入力、右：結果）
- **左エリア**：
  - 英文入力欄（textarea、リサイズ可能）
  - 文字数・単語数カウンター
  - 「添削する」ボタン（ピンクのアクセントカラー）
  - 文体指定セレクタ（ビジネス / カジュアル / 学術）※任意
- **右エリア**：
  - 上段：原文と修正文の並列表示（左右 or 上下切り替え可）
  - 修正箇所はハイライト（観点ごとに色分け：Grammarは赤系、Vocabularyは青系、Toneは緑系など）
  - ハイライトをホバー → ポップアップで修正理由と解説を表示
  - 下段：観点別サマリー（Grammar 3件、Vocabulary 2件…）

### 6.3 添削画面（Challenge Mode）

- Free Mode のレイアウトをベースに、以下を追加：
- **画面上部に「お題」カード**：
  - 例：「Lv.3 関係代名詞を使って、自分の上司を紹介する文章を書いてください（80〜120語）」
  - 使ってほしい文法・構文のヒント表示
- 添削結果に「お題で要求された文法を使えているか」のチェックを追加

### 6.4 履歴ページ

- 左サイド：フィルタ（日付範囲、レベル、観点タグ、モード種別）
- 右メイン：添削エントリのリスト（カード形式）
  - 各カード：日付、最初の一文プレビュー、レベル、主な観点タグ
- カードクリックで添削画面を再表示

### 6.5 マイページ

3つのセクションで構成：
- **① プロフィール / レベル**
  - 現在のレベル、経験値バー、次のレベルまでの距離
  - 連続学習日数（ストリーク）
- **② バッジコレクション**
  - 獲得済みバッジと未獲得バッジをグリッド表示
  - 未獲得は薄く表示、獲得条件をホバーで表示
- **③ 頻出ミスランキング**
  - 観点別タブ（Grammar / Vocabulary / Tone / Structure）
  - トップ10の頻出ミスをバーチャートで表示
  - 項目クリックで該当する過去の添削へジャンプ

## 7. デザインシステム

### 7.1 コンセプト

- **Notionのようなミニマルで落ち着いた白基調**
- アクセントカラーは**薄いピンク**で、優しく親しみやすい学習体験を演出
- 情報密度は適度に高く、ただし余白を十分にとる
- 角丸は中程度（`border-radius: 8〜12px`）

### 7.2 カラーパレット（提案）

| 用途 | カラー | 備考 |
|------|--------|------|
| 背景（ベース） | `#FFFFFF` / `#FAFAFA` | 白ベース |
| テキスト（本文） | `#1F2937` | ダークグレー、真っ黒は避ける |
| テキスト（補助） | `#6B7280` | ミドルグレー |
| ボーダー | `#E5E7EB` | 薄いグレー |
| **アクセント（メイン）** | `#F9A8D4` | 薄いピンク |
| アクセント（濃） | `#EC4899` | ホバー・強調用 |
| アクセント（淡） | `#FCE7F3` | 背景ハイライト用 |
| 成功 | `#10B981` | 正解・達成 |
| 警告 | `#F59E0B` | 文法ミスのハイライト |
| エラー | `#EF4444` | 重要な指摘 |

※Claude Designで提案された色が気に入ればそちらを採用してもOK。

### 7.3 タイポグラフィ

- **英文**：`Inter` または `Source Sans 3`
- **日本語**：`Noto Sans JP`
- ベース：16px、見出しはH1=32px、H2=24px、H3=20px、本文は1.6〜1.75行間

### 7.4 コンポーネント指針

- ボタン：角丸、アクセントピンクは「Primary Action」に限定
- カード：白背景＋薄いボーダー＋控えめなシャドウ
- バッジ：カラフルにしすぎず、モノクロ + アクセントピンクで統一感を保つ
- アイコン：`Lucide` 系のアウトラインアイコン

---

## 8. 技術前提（デザインに影響する範囲）

- 添削エンジン：**Claude API**
  - レスポンス時間は数秒〜10秒程度を想定 → **ローディング状態のUIを必ず用意**（スケルトン or プログレス）
- 初期はローカル/シングルユーザーで動作する想定のため、認証・課金UIは不要

---

## 9. スコープ外（今回は作らない）

- ログイン・ユーザー登録画面
- 料金プラン・決済画面
- モバイル版デザイン（PC版が固まり次第、別途依頼予定）
- ソーシャル機能（他ユーザーとのランキング比較など）
- 音声入力・音声読み上げ

---

## 10. 今回の依頼範囲

1. **PC版のプロトタイプ**を作成してください
2. 上記5画面（トップ / Free Mode添削 / Challenge Mode添削 / 履歴 / マイページ）のデザイン
3. ハイライトや解説ポップアップなど、インタラクション要素の挙動もわかる形で
4. レベルやバッジなど、ゲーミフィケーション要素のビジュアルも含めて提案してほしい

色味・フォント・アイコンの最終的なチューニングはデザイン出力を見てから相談します。
まずは叩き台を出してもらえると助かります。
