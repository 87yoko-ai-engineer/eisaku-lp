## 02 ・ デザイン方針

### 🔬 洗練されたツール感
「勉強アプリ」ではなく「頭のいいプロダクト」。Notionに近い白基調のミニマルさを基本とし、ピンクは差し色として効かせる。

### 📸 実画面を主役に
添削画面・スコア・ハイライトなどの実UI（ダミー）を大きく見せることで、使用感をリアルに伝える。説明文は最小限に。

### ✂️ 余白と情報密度のバランス
詰め込まない。1セクション1メッセージ。セクション間は十分な余白（80〜120px）を確保し、読み疲れさせない。

### 🎮 ゲーミフィケーションはさりげなく
Lv.4バッジ・スコア「82」などは視覚的に見せるが、ゲームっぽさは前に出さない。知性と遊び心のバランスを保つ。

## 06 ・ セクション詳細

### ① Hero — 第一印象で引き込む

- **レイアウト**：中央揃え。ロゴ＋ナビバー → キャッチコピー（大）→ サブコピー（小）→ CTAボタン（ピンク）→ アプリ画面モックアップ（ブラウザ風フレーム）
- **キャッチコピー**：「書いて、直して、伸びていく。あなた専用の英作文コーチ。」
  - 「伸びていく」部分はピンク（`#EC4899`）でアクセント
- **アプリ画面モック**：Free Mode 添削画面（スコア82・ビフォー・アフター表示）を配置
- **背景**：薄いグラデーション（`#FFFFFF` → `#FCE7F3`、上から下へ）

---

### ② 課題提起 — 「こんな悩みありませんか？」

- **背景**：`#FAFAFA`
- **レイアウト**：グリッド 4列（悩みカード4枚）
- **悩み例**：
  1. 「書いても『これで合ってる？』と自信が持てない」
  2. 「知識はあるのに、書くと出てこない」
  3. 「間違いの理由が分からず、同じミスを繰り返す」
  4. 「続けられる仕組みがなく、いつも三日坊主」
- **カード仕様**：白背景・薄ボーダー・Lucide系アウトラインアイコン・shadow-sm

---

### ③ 3つの特徴 — 「EiSakuでできること」

**縦に3段・左右交互レイアウト（テキスト ↔ UIモック）**

| # | タイトル | UIモック |
|---|---------|---------|
| FEATURE 01 | Grammar だけじゃない、多角的な添削 | ビフォー・アフター並列表示 ＋ 解説ノート |
| FEATURE 02 | 今のレベルに合った課題で、着実にステップアップ | Lv.1〜10 一覧・現在地ハイライト |
| FEATURE 03 | 添削履歴で、弱点を丸ごと把握する | 添削履歴リスト（スコア・タグ付き） |

- 奇数セクション：白背景 / 偶数セクション：`#FAFAFA` 背景（交互リズム）

---

### ④ 使い方の流れ — 3ステップ

- **背景**：`#FAFAFA`
- **レイアウト**：横並び 3カード ＋ 矢印区切り

| ステップ | タイトル | UI要素 |
|---------|---------|--------|
| STEP 01 | 英文を書く | テキスト入力エリアのモック（点滅カーソル付き） |
| STEP 02 | AIが添削・解説 | ビフォー（黄系ハイライト）→ アフター（ピンクハイライト） |
| STEP 03 | 振り返って伸びる | スコアの成長グラフ（棒グラフ形式） |

- **ビフォー・アフター例**：
  - Before：`I want visit Paris where has many beautiful buildings...`
  - After：`I want to visit Paris, where there are many beautiful buildings...`

---

### ⑤ 最後のCTA — 決断を促す

- **背景**：ピンク系グラデーション（薄め）
- **コピー**：「今日から、書いて伸びる習慣を。」
- **CTAボタン**：「EiSakuを試してみる →」（ピンク・大）
- **補足**：「✓ 無料でスタート ・ ✓ アカウント登録不要 ・ ✓ いつでも使える」

## 07 ・ コンポーネント仕様

| コンポーネント | 仕様 |
|--------------|------|
| **ナビバー** | 白背景・薄ボーダー下線・sticky固定。左：ロゴ「EiSaku」（ピンクアクセント）、右：テキストリンク＋CTAボタン（小） |
| **CTAボタン（Primary）** | 背景 `#EC4899`・テキスト白・`border-radius: 8px`・`padding: 14px 32px`。ホバー：`#DB2777` |
| **添削UIモック** | 左：原文（取り消し線＋黄系ハイライト）、右：修正文（ピンク系アンダーライン）、スコア：太字数字＋ピンクプログレスバー |
| **観点タグ** | Grammar：赤系 / Vocabulary：青系 / Tone：緑系 / Structure：紫系。`border-radius: 4px`・小フォント |
| **課題提起カード** | 白背景・`border: 1px solid #E5E7EB`・`border-radius: 10px`・アイコン＋テキスト・shadow-sm |
| **レベルバッジ** | 達成済み：グレー / 進行中：ピンク（Dark）/ 未到達：グレーアウト |

## 08 ・ インタラクション

| インタラクション | 実装方法 |
|---------------|---------|
| **スクロールフェードイン** | 各セクション：初期 `opacity:0 / translateY(24px)` → 画面内進入で `visible` クラス付与（0.6s ease-out）。`IntersectionObserver` で実装 |
| **スコアカウントアップ** | Hero内スコア「82」：表示時に0→82へカウントアップ（1.0s、cubic-bezier） |
| **プログレスバーアニメーション** | スコアバーが0%→82%へ伸びるアニメーション（1.2s） |
| **ビフォー・アフターハイライト** | 原文・修正文のハイライト色でビジュアル差分を表現 |
